# 位移-应变-应力-能量(psie_active)链路与相场耦合说明（2.1main.i / 2.1_Sub.i）

## 1. 输入文件的“调用顺序”怎么理解

MOOSE 的输入文件不是“从上到下顺序执行脚本”，而是：

1. 解析输入文件，创建对象（变量、材料、核、边界条件、MultiApp、Transfer 等）。
2. 在每个时间步/非线性迭代中，按对象类型与依赖关系执行：
   - 先更新材料（Material）得到应力、应变、能量等材料属性（MaterialProperty）。
   - 再由核（Kernel）装配方程残差/雅可比，例如力平衡用 StressDivergenceTensors，热传导用 HeatConduction。
   - MultiApp/Transfer 在它们配置的 execute_on 时刻触发，用于主从耦合数据交换。

因此“调用顺序”更准确地说是：谁依赖谁（比如 Kernel 依赖材料属性 stress），以及它们在每个时间步的触发时机（execute_on）。

## 2. 重点：[Physics] 103-106 行到底在干什么

对应文件：[2.1main.i:L103-L117](file:///home/yp/projects/reproduction/1Tasks/Rod/s6GPlaneStrain/s5NewFracture/2.1main.i#L103-L117)

这一段声明的是固体力学“物理块”：

- `strain = SMALL`：小变形应变（线性化几何），应变来自位移梯度的对称部分。
- `incremental = true`：增量形式（每步用增量更新应变/内部变量），与蠕变等历史变量更匹配。
- `planar_formulation = GENERALIZED_PLANE_STRAIN` + `scalar_out_of_plane_strain = scalar_strain_zz`：
  - 2D 网格上引入一个标量自由度 `scalar_strain_zz` 来表示“厚度方向平均应变”，从而实现广义平面应变（等价于把 3D 的一个应变分量用标量变量承载）。
- `eigenstrain_names = 'thermal_eigenstrain creep_eigenstrain'`：
  - 指定“本征应变”（应力自由应变）列表。后续机械应变一般按
    \[
      \varepsilon^{mech} = \varepsilon^{total} - \varepsilon^{thermal} - \varepsilon^{creep}
    \]
    进入本构。
- `use_automatic_differentiation = false`：该力学链路使用非 AD（Real）材料/核。

## 4. psie_active 为什么“没有正常积累”

### 4.1 psie_active 在代码里是什么

本算例的 `psie_active` 来自材料 `IsotropicElasticityNonAD`：

- 输入文件位置：[2.1main.i:Elasticity](file:///home/yp/projects/reproduction/1Tasks/Rod/s6GPlaneStrain/s5NewFracture/2.1main.i#L495-L508)
- 代码位置：[IsotropicElasticityNonAD.C](file:///home/yp/projects/reproduction/src/NewCreep/IsotropicElasticityNonAD.C)

它计算的是“拉伸部分的弹性能”（根据 `decomposition = SPECTRAL` 取正部分），并可选用历史最大化（不可逆性）：

\[
H(t)=\max_{\tau\le t}\psi_e^+(\tau)
\]

### 4.2 根因：use_history_max 原来是 false

在原输入里 `use_history_max = false`，意味着：

- `psie_active` 只是“当前时刻的 \(\psi_e^+\)”；
- 在卸载/应力回落时它会下降，看起来就像“不积累”。

已将其改为 `use_history_max = true`，使 `psie_active` 变成历史最大值（不可逆积累）。

## 5. 主-子相场耦合：为什么还需要把 psie_active / a1 做成变量

本算例采用 MultiApp：

- 主程序执行固-热-化学，得到 `psie_active`、`a1`、`stress_I`；
- 子程序求解相场变量 `d`，其自由能里包含 `g(d)*psie_active`：
  - 子程序位置：[2.1_Sub.i:psi](file:///home/yp/projects/reproduction/1Tasks/Rod/s6GPlaneStrain/s5NewFracture/2.1_Sub.i#L159-L168)

Transfer 传的是“变量场”，不是材料属性。因此主程序里必须存在同名的变量：

- 子程序 AuxVariables 已定义 `psie_active/a1/stress_I`：[2.1_Sub.i](file:///home/yp/projects/reproduction/1Tasks/Rod/s6GPlaneStrain/s5NewFracture/2.1_Sub.i#L75-L90)
- 主程序之前缺少 `psie_active/a1` 变量，导致无法把主程序材料计算的量稳定地传给子程序。

已在主程序中增加：

- AuxVariables：`psie_active` 与 `a1`
- AuxKernels：用 `MaterialRealAux` 把同名材料属性拷贝到变量（psie_active 每步更新，a1 初始赋值即可）

对应修改位置：[2.1main.i](file:///home/yp/projects/reproduction/1Tasks/Rod/s6GPlaneStrain/s5NewFracture/2.1main.i)
